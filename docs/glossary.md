# 用語集（まずはここから）

GitHubでよく出てくる言葉を、**身近なたとえ**で説明します。
完璧に覚える必要はありません。「こんなイメージ」とつかめれば十分です。

!!! tip "読み方"
    上から順に、だんだん使う言葉が出てきます。分からない言葉が出てきたら、このページに戻ってきてください。

## :material-book-open-variant: 基本の言葉

### :material-folder-outline: リポジトリ（Repository / 略して **repo**）
**ファイルを入れておく「フォルダ（保管箱）」**のことです。
プロジェクトや資料ごとに1つ作ります。

> 📁 たとえ：「○○プロジェクト」という名前の箱を1つ用意するイメージ。

!!! note "「repo」って略すの？"
    はい。英語圏では「リポジトリ」を **repo（レポ）** と略すのが一般的です。
    画面やツールでも `repo` と表示されることがあるので、覚えておくと迷いません。

### :material-content-save-outline: コミット（Commit）
**変更を記録して保存すること**です。「ここまでの作業に名前をつけて記録」という感覚。
コミットには「何を変えたか」のひとことメモ（コミットメッセージ）を付けます。

> 💾 たとえ：ゲームの「セーブポイント」。あとからそのポイントに戻れる。

### :material-cloud-upload-outline: プッシュ（Push）
**自分のPCで記録した変更を、GitHub（ネット上）にアップロードすること**です。

> ⬆️ たとえ：手元で書いた書類をクラウドにアップして、みんなが見られる状態にする。

### :material-cloud-download-outline: プル（Pull）
**GitHub（ネット上）にある最新の状態を、自分のPCに取り込むこと**です。プッシュの逆方向。

> ⬇️ たとえ：クラウド上の最新版を手元にダウンロードする。

### :material-content-copy: クローン（Clone）
**GitHub上のリポジトリを丸ごと自分のPCにコピーする** ことです。手元で作業を始める「最初の1回」の操作。

> 📋 たとえ：共有棚のファイル一式を、自分の机にまるごとコピーして持ってくる。

### :material-cloud-outline: リモート / origin（オリジン）
**GitHub上にあるリポジトリ**（手元のコピーに対する「向こう側」）のことです。`origin` は、その相手にGitがふつうに付ける **呼び名**。プッシュ・プルの相手先になります。

> ☁️ たとえ：手元のノート（ローカル）に対する、クラウド上の“正本”。

## :material-account-group: 共同作業の言葉

### :material-source-branch: ブランチ（Branch）
**作業用の「分かれ道」**です。本体（みんなが使う版）に影響を与えずに、別の場所で安心して作業できます。

> 🌿 たとえ：清書用のノート（本体）とは別に、下書き用のノートを用意するイメージ。

### :material-source-merge: マージ（Merge）
**ブランチでの作業を本体に合流させること**です。下書きが完成したら、清書に反映する感覚。

> 🔀 たとえ：下書きノートの内容を、清書ノートにまとめる。

### :material-alert-outline: コンフリクト（Conflict／競合）
**同じ場所を別々に変更したため、どちらを採用するか自動で決められない状態** です。マージやプルのときに起きます。

> ⚡ たとえ：2人が同じ行を別々に書き直して、「どっちを残す？」と聞かれる状態。

!!! tip "起きても慌てない"
    コンフリクトは“事故”ではなく、よくあることです。Claudeに「コンフリクトを直して」と頼んだり、詳しい人に相談すればOKです。

### :material-source-pull: プルリクエスト（Pull Request / 略してプルリク・PR）
**「この変更を本体に入れていいですか？」という確認・相談の依頼**です。
ほかの人に見てもらってから（レビュー）、マージするのが一般的です。

> 🙋 たとえ：「この修正でOKですか？」と上司に確認してもらってから清書に反映する。

### :material-file-tree: ワークツリー（Worktree）
**同じリポジトリ（repo）を、複数の作業場所（フォルダ）に同時に開ける仕組み**です。
ふだんはブランチを切り替えると“いまいる場所”が入れ替わりますが、ワークツリーを使うと「ブランチAのフォルダ」と「ブランチBのフォルダ」を別々に持って、行き来せずに同時に作業できます。

> 🌲 たとえ：1冊のノート（repo）に対して、下書き用の机と清書用の机を別々に用意して、両方を開きっぱなしにするイメージ。

!!! note "これは少し上級の話"
    最初は使わなくても大丈夫です。「2つのブランチを同時に開きたい」と思ったときの選択肢、と覚えておけば十分。
    くわしくは [もっと使う（上級編）](advanced.md) を参照してください。

### :material-undo-variant: リバート（Revert）
**ある変更を「打ち消す」ための、新しいコミットを作ること**です。
過去を消すのではなく、「ここは取り消しました」という記録を新たに足すので、**履歴が残り、比較的安全**です。

> ↩️ たとえ：書き間違えた行を消しゴムで消すのではなく、「さっきのは無しで」と新しい一行を足して訂正するイメージ。

### :material-history: リベース（Rebase）
**コミットの並びを整理し直す（土台を付け替える）こと**です。
履歴をきれいに見せられますが、**履歴を書き換える**ため、扱いには注意が必要です。

> 🧱 たとえ：積んだブロックを一度はずして、別の土台に積み直すイメージ。

!!! warning "リベースは共有ブランチで使わない"
    みんなで使うブランチ（`main` など）でのリベースは、混乱のもとになります。
    基本は **マージ**（上の項目）でOK。リベースは自分だけのブランチで、共有する前の整理に使う程度に。
    どちらも、くわしい使い方は [もっと使う（上級編）](advanced.md) を参照してください。

## :material-trending-up: 一歩進んだ言葉

最初は読み飛ばしてOK。慣れてきたら戻ってきてください。

### :material-file-compare: 差分（diff／ディフ）
**変更の前後で「どこが・どう変わったか」** を示したものです。コミットやプルリクで「何を変えたか」を確認するときに見ます。

> 🔍 たとえ：文書の校正で、追加した行・消した行に色が付いて見えるイメージ。

### :material-download-outline: フェッチ（Fetch）
**GitHub上の最新を「取得だけ」する** ことです。プルが「取得＋手元へ取り込み」なのに対し、フェッチは取ってくるだけ（中身を見てから取り込みたいときに使う）。

> 📥 たとえ：届いた荷物を玄関まで取りに行く（フェッチ）／部屋に運んで開封まで（プル）。

### :material-source-fork: フォーク（Fork）
**他人のリポジトリを、自分のアカウントにまるごとコピーする** ことです。元を変えずに自分用に持てるので、ほかの人のプロジェクトに参加するときなどに使います。

> 🍴 たとえ：他人のレシピを自分のノートに書き写して、自由に手を加える。

### :material-archive-outline: スタッシュ（Stash）
**作業の途中を「一時的にしまっておく」** 機能です。別の作業を割り込ませたいとき、いまの変更を退避できます。

> 🗄️ たとえ：作りかけの書類を引き出しに一旦しまい、急ぎの用事を片付けてから出して続ける。

### :material-eye-off-outline: .gitignore（ギットイグノア）
**「記録しない（管理しない）ファイル」を指定する設定ファイル** です。一時ファイルや、見せたくない情報を誤って共有しないために使います。

> 🚫 たとえ：「これは記録しなくていい」リスト。

!!! warning "パスワード等は必ず除外"
    パスワード・APIキー・個人情報を含むファイルは、`.gitignore` に入れて **共有しない** ようにしましょう。

### :material-map-marker: HEAD（ヘッド）
**いま自分が「どのコミット／ブランチにいるか」を指す目印** です。

> 📍 たとえ：地図の「現在地」マーカー。

### :material-tag-outline: タグ／リリース（Tag / Release）
**特定の時点に名前（`v1.0` など）を付けて、節目として記録・公開する** ことです。

> 🏷️ たとえ：本に「初版」「第2版」と版を付けるイメージ。

### :material-cog-outline: GitHub Actions（ギットハブ・アクションズ）
**プッシュなどをきっかけに、決めた処理（チェックや公開など）を自動で実行する仕組み** です（CI/CD と呼ばれます）。

> 🤖 たとえ：「プッシュしたら自動でチェックする」といった流れ作業を組める。

## :material-dots-horizontal-circle-outline: その他よく見る言葉

| アイコン | 言葉 | かんたんな意味 |
|:---:|---|---|
| :material-file-document-outline: | **README（リードミー）** | リポジトリの「説明書・表紙」になるファイル |
| :material-note-outline: | **Issue（イシュー）** | やること・課題・バグなどを記録しておく付箋のようなもの |
| :material-web: | **GitHub Pages** | GitHubのファイルをWebサイトとして無料公開できる仕組み（この資料もこれで公開） |

!!! note "Git と GitHub の違い"
    - :material-git: **Git** … 変更履歴を管理する「仕組み・道具」そのもの
    - :material-github: **GitHub** … その仕組みをWebでみんなが使えるようにした「サービス・置き場所」

    まずは「GitHub＝みんなで使えるファイル置き場＋履歴管理」と覚えておけば大丈夫です。

言葉のイメージがつかめたら、次は[アカウント作成と初期設定](account-setup.md)に進みましょう。
