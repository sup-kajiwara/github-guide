# リポジトリの名前の付け方（命名規則）

!!! info "このページのゴール"
    新しくリポジトリ（repo）を作るときの **名前の付け方の決まり** をそろえること。
    名前を統一すると、**探しやすく・並びがきれいに・取り違えにくく** なります。

<figure markdown="span">
  ![ファイルに名前を付けて整理するイメージ](images/illustrations/add-files.svg){ width="300" }
  <figcaption>名前のルールを決めておくと、毎回「どう付ける？」で迷いません</figcaption>
</figure>

!!! tip "なぜ名前のルールが必要？"
    **結論：あとから探す人（自分を含む）のためです。**
    リポジトリが増えると、名前がバラバラだと一覧から目当ての箱を見つけにくくなります。
    ルールをそろえておけば、**名前を見ただけで「何の箱か」が分かり**、並びもきれいになります。

---

## 1. 基本ルール

| 区分 | 形式 | 例 |
| :--- | :--- | :--- |
| 案件リポジトリ | `pj-<顧客略称>-<システム>` | `pj-acme-attendance` |
| その他リポジトリ | `<接頭辞>-<内容>`（kebab-case） | `tool-invoice-export` |

- :material-format-letter-case-lower: **すべて小文字・ハイフン区切り**（kebab-case）。
- :material-account-question: **顧客略称は英字**。実名フル表記は **セキュリティの観点で避ける**。
- :material-text-short: **システムは短い語**（下の[対訳表](#4)に準拠）。

!!! warning "実在の顧客名・案件名をそのまま使わない"
    リポジトリ名は **URLや一覧に残る** ため、`pj-` の顧客略称に **実在の会社名フル表記や個人名** を入れないでください。
    社内で通じる **短い英字の略称** を決めて使います（例：`acme` `sample` など）。
    判断に迷う場合は [AIと安全に付き合う](ai-safety.md) も参照してください。

---

## 2. 接頭辞（prefix）一覧

**結論：先頭の2〜3文字で「何の箱か」を表します。** まずは下の4つを覚えれば十分です。

| 接頭辞 | 用途 |
| :--- | :--- |
| `pj-` | 顧客案件（プロジェクト）リポジトリ |
| `tool-` | 社内向けツール・自動化スクリプト |
| `docs-` | ドキュメント・ナレッジ（コード以外） |
| `tpl-` | テンプレート・雛形リポジトリ |

??? note "状況に応じて使う接頭辞（基本は使いません）"
    最初は気にしなくてOK。必要になったときに見てください。

    | 接頭辞 | 用途 |
    | :--- | :--- |
    | `lib-` | 共通ライブラリ・部品（再利用前提） |
    | `skill-` | Claude Code のスキル／プラグイン |
    | `infra-` | インフラ・サーバ構成・IaC |
    | `poc-` | 検証・試作（PoC、使い捨て前提） |
    | `sandbox-` | 個人の実験・学習用 |
    | `archive-` | 運用終了・凍結したリポジトリ |

---

## 3. 命名例

```text
pj-acme-attendance
pj-sample-inventory
pj-example-sales
pj-acme-order
```

!!! note "顧客略称はダミーです"
    上の `acme` / `sample` / `example` は **説明用の架空の略称** です。
    実際の案件では、社内で取り決めた略称に置き換えてください（実在名のフル表記は避ける）。

## 5. 運用ルール（ブレ防止）

- :material-numeric-1-circle: **単数形・小文字に統一**（`sales` であって `Sales`／`saleses` にしない）。
- :material-numeric-2-circle: **この表を「正」とし、新語が出たら追記**（複数の訳が浮かんだら必ずどちらか1つに固定）。
- :material-numeric-3-circle: 迷ったら表の左列で **社内検索 → 右列を採用**。

!!! tip "AIに名前を考えてもらうときは"
    Claudeに「このリポジトリ名でいい？」と聞くと、kebab-caseの整形や英語表記の候補を出してくれます。
    そのときも **この対訳表を見せて「この表に合わせて」** と伝えると、社内ルールからブレません。
