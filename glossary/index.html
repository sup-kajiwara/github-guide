    用語集（まずはここから）¶

    GitHubでよく出てくる言葉を、身近なたとえで説明します。 完璧に覚える必要はありません。「こんなイメージ」とつかめれば十分です。

    読み方

    上から順に、だんだん使う言葉が出てきます。分からない言葉が出てきたら、このページに戻ってきてください。

    基本の言葉¶

    リポジトリ（Repository / 略して repo）¶

    ファイルを入れておく「フォルダ（保管箱）」のことです。 プロジェクトや資料ごとに1つ作ります。

    📁 たとえ：「○○プロジェクト」という名前の箱を1つ用意するイメージ。

    「repo」って略すの？

    はい。英語圏では「リポジトリ」を repo（レポ） と略すのが一般的です。 画面やツールでも repo と表示されることがあるので、覚えておくと迷いません。

    コミット（Commit）¶

    変更を記録して保存することです。「ここまでの作業に名前をつけて記録」という感覚。 コミットには「何を変えたか」のひとことメモ（コミットメッセージ）を付けます。

    💾 たとえ：ゲームの「セーブポイント」。あとからそのポイントに戻れる。

    プッシュ（Push）¶

    自分のPCで記録した変更を、GitHub（ネット上）にアップロードすることです。

    ⬆️ たとえ：手元で書いた書類をクラウドにアップして、みんなが見られる状態にする。

    プル（Pull）¶

    GitHub（ネット上）にある最新の状態を、自分のPCに取り込むことです。プッシュの逆方向。

    ⬇️ たとえ：クラウド上の最新版を手元にダウンロードする。

    共同作業の言葉¶

    ブランチ（Branch）¶

    作業用の「分かれ道」です。本体（みんなが使う版）に影響を与えずに、別の場所で安心して作業できます。

    🌿 たとえ：清書用のノート（本体）とは別に、下書き用のノートを用意するイメージ。

    マージ（Merge）¶

    ブランチでの作業を本体に合流させることです。下書きが完成したら、清書に反映する感覚。

    🔀 たとえ：下書きノートの内容を、清書ノートにまとめる。

    プルリクエスト（Pull Request / 略してプルリク・PR）¶

    「この変更を本体に入れていいですか？」という確認・相談の依頼です。 ほかの人に見てもらってから（レビュー）、マージするのが一般的です。

    🙋 たとえ：「この修正でOKですか？」と上司に確認してもらってから清書に反映する。

    その他よく見る言葉¶

    アイコン 言葉 かんたんな意味
    クローン（Clone） GitHub上のリポジトリを丸ごと自分のPCにコピーすること
    README（リードミー） リポジトリの「説明書・表紙」になるファイル
    Issue（イシュー） やること・課題・バグなどを記録しておく付箋のようなもの
    GitHub Pages GitHubのファイルをWebサイトとして無料公開できる仕組み（この資料もこれで公開）

    Git と GitHub の違い

    • Git … 変更履歴を管理する「仕組み・道具」そのもの
    • GitHub … その仕組みをWebでみんなが使えるようにした「サービス・置き場所」

    まずは「GitHub＝みんなで使えるファイル置き場＋履歴管理」と覚えておけば大丈夫です。
