    変更を記録する（コミット）¶

    この章のゴール

    ファイルを変更して、その変更を 記録（コミット） し、GitHubに反映（プッシュ） できるようになること。

    flowchart LR
    A[ファイルを<br/>変更する] --> B[コミット<br/>変更を記録 💾]
    B --> C[プッシュ<br/>GitHubへ反映 ⬆️]
    C --> D([みんなが見られる状態 🎉])

    1. コミットとは（おさらい）¶

    コミットは、ゲームの「セーブポイント」のようなものです（→ 用語集）。 「ここまでの変更に名前をつけて記録する」のがコミット。あとからその時点に戻れます。

    そして プッシュ は、そのセーブを GitHub（ネット上）にアップロード することです。

    コミットとプッシュの関係

    • コミット … 手元で「記録」する（まだ自分だけ）
    • プッシュ … その記録を「GitHubに送る」（みんなに共有）

    2. 変更してコミット・プッシュする¶

    ためしに、前章で作ったファイルを少し書き換えて、記録してみましょう。

    メモ.txt に「2行目を追加しました」と1行足して、
「メモを追記」というメッセージで記録（コミット）して、GitHubに反映して。

    Claudeが「コミット → プッシュ」まで進めてくれます。 どんなメッセージにするか迷ったら、それも相談できます。

    1. VSCodeでファイルを編集して 保存（Ctrl/⌘ + S）
    2. 左側の ソース管理（枝アイコン ） を開く
    3. 変更したファイルが一覧に出るので、＋ で「ステージ」する
    4. 上の入力欄に メッセージ（例：メモを追記）を書く
    5. ✓ コミット を押す
    6. 同期（Sync Changes） または ... → プッシュ でGitHubへ反映

    📷 画面キャプチャ枠（あとで差し込み）

    VSCodeのソース管理パネルの画面を入れます。 ![ソース管理パネル](images/commit-01-scm.png){ width="700" }

    「ステージ」ってなに？

    「今回の記録に、この変更を含めますよ」と印をつける作業です。 難しく考えず、記録したいファイルの＋を押す とだけ覚えればOKです。

    3. 良いコミットメッセージの書き方¶

    コミットメッセージは「何をしたか」をひとことで書きます。後で履歴を見たときに分かりやすくなります。

    わかりにくい例 わかりやすい例
    更新 表紙に説明文を追加
    修正 日付の誤りを修正（4月→5月）
    いろいろ 問い合わせ先の電話番号を更新

    コツは「〜を〜した」

    「何を」「どうした」 が分かれば十分です。 例：タイトルを変更、画像を3枚追加、誤字を修正

    メッセージが思いつかないときは

    Claudeに「いまの変更に合うコミットメッセージを考えて」と頼めます。 変更内容を見て、短いメッセージを提案してくれます。
